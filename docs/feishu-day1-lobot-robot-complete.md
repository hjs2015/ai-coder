### 7. 本地 Embedding 记忆方案（第 12 章核心内容）

#### 7.1 模型信息
- **模型名称**: `hf:ggml-org/embeddinggemma-300M-GGUF/embeddinggemma-300M-Q8_0.gguf`
- **模型大小**: 约 0.6 GB
- **作用**: 把文字变成"可比较距离的数字向量"，用于记忆检索，不负责生成最终回答

#### 7.2 工作机制
流程：**先入库，再检索**

**入库阶段（索引）**:
- 文本 → Embedding 模型 → 向量
- 向量存储到向量数据库

**检索阶段（memory_search）**:
- 用户查询 → Embedding 模型 → 查询向量
- 查询向量 vs 库存向量 → 相似度匹配
- 返回最相关的记忆片段

### 8. Memory 记忆功能最佳实践（第 11 章）

#### 8.1 清晰划分长期记忆 vs 每日记忆

**MEMORY.md**:
- 只放"几个月后回头看仍然有价值"的内容
- 长期记忆、核心知识、关键决策

**memory/YYYY-MM-DD.md**:
- 记录每天的过程、现场上下文
- 临时信息、日常对话、过程记录

**提炼机制**:
- 时不时从当日 Memory 中"提炼"重要信息
- 整理进 MEMORY.md

#### 8.2 优先让 Agent 自己写 Memory

**建议用自然语言驱动，而不是手动编辑文件**:
- "把刚才排查这个 Bug 的过程记到今天的 memory 文件。"
- "把这个项目的关键信息写入 MEMORY.md，后面要能搜到。"

**优势**:
- Agent 会通过自己的工具链切块和写入
- 对索引更加友好
